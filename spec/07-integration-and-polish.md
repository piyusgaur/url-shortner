# Step 7: Integration and Polish

## Goal
Connect the frontend to the backend cleanly and make the experience feel finished.

## What to build
- Wire the frontend to the backend API.
- Add input validation and friendly error messages.
- Improve layout, spacing, and visual hierarchy.
- Confirm redirect behavior works from the shortened link.

## What to avoid
- Do not add unnecessary features that distract from the take-home.
- Do not sacrifice clarity for visual flair.

## Done when
- Frontend and backend work together end to end.
- The UI feels polished enough for a review.
- The main user flow is easy to demonstrate.

## Suggested commit
- `feat(frontend): connect api and improve user experience`
